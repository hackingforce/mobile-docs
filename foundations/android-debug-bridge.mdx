# Android Debug Bridge
O Android Debug Bridge (ADB) é uma ferramenta de linha de comando que permite a comunicação entre um dispositivo Android e um computador. O ADB é parte do Android SDK e é utilizado para depurar aplicativos, instalar APKs, transferir arquivos, e muito mais.

> [!INFO]
> adb está incluído no pacote Android SDK Platform Tools.


## Dispositivos Ativos
Para listar os dispositivos ativos conectados ao computador, execute o seguinte comando:

```bash filename="Terminal"
adb devices
```

## Transferência de Arquivos
Para transferir arquivos entre o computador e o dispositivo, utilize o comando `push` para enviar arquivos do computador para o dispositivo e `pull` para baixar arquivos do dispositivo para o computador.

Para enviar um arquivo para o dispositivo, execute o seguinte comando:

```bash filename="Terminal"
adb push arquivo.txt /sdcard/
```

Para baixar um arquivo do dispositivo, execute o seguinte comando:

```bash filename="Terminal"
adb pull /sdcard/arquivo.txt
```

## Redirecionamento de Portas
O ADB permite redirecionar portas entre o dispositivo e o computador. Isso é útil para acessar serviços em execução no dispositivo, como um servidor web.

Para redirecionar a porta `8080` do dispositivo para a porta `8080` do computador, execute o seguinte comando:

```bash filename="Terminal"
adb forward tcp:8080 tcp:8080
```

Para fazer o redirecionamento de portas em sentido contrário, execute o seguinte comando:

```bash filename="Terminal"
adb reverse tcp:8080 tcp:8080
```

`--remove` pode ser usado para remover as portas redirecionadas.

## Interromper o servidor do adb
Em alguns casos, pode ser necessário encerrar o processo do servidor do adb e reiniciá-lo para resolver o problema. Por exemplo, esse pode ser o caso quando adb não responde a um comando.

Para interromper o servidor do adb, use o comando a baixo. Em seguida, é possível emitir qualquer outro comando do adb para reiniciar o servidor.

```bash filename="Terminal"
adb kill-server
```
